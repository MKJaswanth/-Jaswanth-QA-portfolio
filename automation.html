Methodology

A testing practice built around risk, reproducibility, and maintainability.

01

Requirement decomposition

Convert flows into clear scenarios with preconditions, test data, expected behavior, priority, and severity. I start with auth, checkout, and data-entry paths because those failures create the highest user-facing risk.

02

Manual and exploratory execution

Run functional, regression, UI, negative, boundary, responsive, and edge-case checks across web products. Exploratory passes catch the gaps scripted checks often miss.

03

Defect evidence

Document exact reproduction steps, environment details, expected versus actual results, and client-facing impact. A useful report reduces developer guesswork.

04

Automation that earns its keep

Use Playwright, Python, pytest, Page Object Model, fixtures, parametrization, dotenv config, HTML reports, and GitHub Actions CI.
