Scope

Covered homepage, navigation, contact actions, cart access, product paths, CTAs, and buyer journey flows for a wholesale commerce experience.

Approach

Prioritized release-risk scenarios first: access control, conversion paths, broken links, form validation, and expected-versus-actual behavior.

What I Found

Cart access without login, incorrect CTA routing, product path failures, and static contact elements that should have opened phone or email actions.

Business Impact

Clear defect evidence helped separate cosmetic issues from risks that affect buyer trust, lead conversion, and release readiness.

Methodology

A testing practice built around risk, reproducibility, and maintainability.

01

Requirement decomposition

Convert flows into clear scenarios with preconditions, test data, expected behavior, priority, and severity.

02

Manual and exploratory execution

Run functional, regression, UI, negative, boundary, responsive, and edge-case checks across web products.

03

Defect evidence

Document exact reproduction steps, environment details, expected versus actual results, and client-facing impact.

04

Automation that earns its keep

Use Playwright, Python, pytest, Page Object Model, fixtures, parametrization, dotenv config, HTML reports, and GitHub Actions CI.
